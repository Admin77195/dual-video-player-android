========================================
  🚀 手动触发 GitHub Actions 构建
========================================

当前问题：Actions 页面显示"开始使用 GitHub Actions"
原因：构建从未运行过，需要手动触发

========================================
  ✅ 解决方案1：启用 Actions（如果被禁用）
========================================

【步骤1】检查 Actions 是否启用
-----------------------------------
1. 在您的 GitHub 仓库页面
2. 点击顶部的 "Settings"（设置）
3. 左侧菜单找到 "Actions" → "General"
4. 查看 "Actions permissions" 部分
5. 确保选择了：
   ✅ "Allow all actions and reusable workflows"
   或
   ✅ "Allow [your organization] actions and reusable workflows"

【步骤2】保存设置
-----------------------------------
1. 如果更改了设置，点击底部 "Save" 按钮
2. 返回 Actions 页面


========================================
  ✅ 解决方案2：手动触发构建
========================================

【方法A】通过 Actions 页面触发
-----------------------------------
1. 点击仓库顶部的 "Actions" 标签
2. 左侧应该能看到 "Build Android APK" 工作流
   （如果看不到，说明 Actions 未启用，请先执行解决方案1）
3. 点击 "Build Android APK"
4. 右侧会出现 "Run workflow" 按钮（绿色）
5. 点击 "Run workflow"
6. 在弹出框中：
   - Branch: 选择 "main" 或 "master"
   - 点击绿色的 "Run workflow" 按钮
7. 等待几秒钟，页面会显示新的运行记录

【方法B】通过推送代码触发
-----------------------------------
如果方法A不行，可以通过推送代码触发：

1. 在本地项目文件夹中
2. 双击运行 "上传到GitHub.bat"
3. 或者手动执行 git push
4. 推送后 Actions 会自动触发


========================================
  ✅ 解决方案3：修改工作流文件触发
========================================

如果上述方法都不行，可以修改工作流文件：

【步骤1】编辑 README.md
-----------------------------------
1. 在 GitHub 仓库页面
2. 点击 "README.md" 文件
3. 点击右上角的铅笔图标（编辑）
4. 在文件末尾添加一个空行或任意文字
5. 点击底部 "Commit changes"
6. 这会触发一次 push，从而触发 Actions

【步骤2】查看 Actions
-----------------------------------
1. 返回 Actions 页面
2. 应该能看到新的构建开始运行


========================================
  🎯 推荐操作顺序
========================================

【第1步】检查 Actions 权限（1分钟）
Settings → Actions → General → 确保已启用

【第2步】手动触发构建（1分钟）
Actions → Build Android APK → Run workflow

【第3步】等待构建完成（20-30分钟）
查看构建进度，等待绿色勾号

【第4步】下载 APK
构建完成后，在 Artifacts 部分下载


========================================
  📱 详细操作：如何手动触发
========================================

【当前您的页面】
您现在看到的是 Actions 的欢迎页面，显示：
- "开始使用 GitHub Actions"
- 一些推荐的工作流模板
- 但没有您的构建记录

【您需要做的】
1. 不要点击页面上的模板
2. 查看页面左侧，应该有一个侧边栏
3. 侧边栏标题是 "All workflows"
4. 下面应该列出 "Build Android APK"

【如果左侧没有侧边栏】
说明工作流文件可能有问题，或者 Actions 未启用


========================================
  🔧 故障排查
========================================

【问题1】左侧看不到 "Build Android APK"
-----------------------------------
原因：
- Actions 未启用
- 工作流文件路径错误
- 工作流文件语法错误

解决：
1. 检查 Settings → Actions 是否启用
2. 确认文件路径：.github/workflows/build.yml
3. 检查 build.yml 语法是否正确

【问题2】点击 Actions 一直是欢迎页面
-----------------------------------
原因：
- 从未运行过任何工作流
- Actions 刚刚启用

解决：
- 手动触发一次构建
- 或者推送一次代码

【问题3】"Run workflow" 按钮是灰色的
-----------------------------------
原因：
- 没有权限
- 不是仓库所有者

解决：
- 确保您是仓库的所有者
- 或者有 Actions 的写入权限
